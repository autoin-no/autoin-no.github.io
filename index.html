Themes for process use:

The AutoIn theme defines three additional color themes for usage with processes:

  • btn/badge[-outline]-planned - An process that has been planned, but not sent to production yet
  • btn/badge[-outline]-start - An process that can be started
  • btn/badge[-outline]-stop - An process that is in progress and can be stopped
  • btn/badge[-outline]-done - An process that is done

NOTE: Use standard Boostrap HTML for badges and buttons

These color themes are using a official way to extend bootstrap so you can use any kind of Boostrap html just with the themes classes changed for the once above (See below for examples). Should be used for badges and buttons as shown with examples below, please don't use for alerts as alerts should convey rather severity like danger/warning/info/sucess after some action.

For more examples of when to use outline or not see below, for when to use process theme styles: Only for buttons to start/stop product processes.

Buttons for planned tasks (if shown instead of hidden), will always have to be in disabled state. For badges it makes more sense to show them, see below.

Selects in different forms:

Selects are to be used when there are predefined options user should pick between, prefer the search version if there can become many options.

For sitations where user should be able to add options on the fly, we'll in the future use datalists.

TextArea for bulk copy/paste:

This component consists of two parts, the underlying textarea with .bc-area with javascript logic handling parsing the copy paste content from excel/word/email sources, and a modal with id #ExcelModal which has custom callback on data-onpasted property where you'll get the pasted data both as a multi-dimensional array and as a string.

The formats supported to be pasted into the text area is:
- Excel: 1 or 2 columns, first must be reg.nr second can optionally be car type (model)
- Word/Text: Either list of reg.nr (bullet points or just each reg.nr on a new line), or just a comma separated list. In all cases car type can optionally be there as well by means of separation either with newline or tab from the reg.nr
